# Редактиране на програмата (Schedule)

Програмата вече се управлява динамично чрез **Firebase Realtime Database** и **Административния панел** (`admin.html`).

**Старият метод с файла `schedule-data.js` вече НЕ се използва.**

## Как да редактирате програмата

1. Влезте в **Admin Panel** (`/admin.html`).
2. Използвайте секцията **Schedule Data (JSON)** или **Visual Schedule Editor**.
   - **Load from Firebase:** Зарежда текущото разписание от базата данни.
   - **Visual Editor:** Позволява визуално редактиране на седмиците и събитията.
   - **Save JSON:** Запазва промените директно в Firebase.

## Структура на данните (JSON)

Данните в Firebase следват следната структура:

1.  **`semesters`**: Дефиниция на семестрите (зимен, летен), ваканции и изпитни сесии.
2.  **`exams`**: Списък с изпити и дати.
3.  **`templates`**: Основното (повтарящо се) разписание за всеки семестър (winter, summer).
4.  **`overrides`**: Изключения за конкретни седмици (напр. преместени часове, колоквиуми, отменени занятия).
5.  **`weeks`**: Списък с началните дати на всяка учебна седмица.

### Примерна структура:

```json
{
  "semesters": {
    "winter": { ... },
    "summer": { ... }
  },
  "exams": [ ... ],
  "templates": {
    "winter": [ ... ],
    "summer": [ ... ]
  },
  "overrides": {
    "3": [ ... ] // Промени за 3-та седмица
  },
  "weeks": [ ... ]
}
```

Всички промени се отразяват мигновено за всички потребители след запазване.
